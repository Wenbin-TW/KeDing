# Data Engineering & Automation Portfolio

本倉庫收錄了我在企業內部主導開發的數據管線、自動化決策系統與爬蟲架構開發紀錄。
** 為了遵守前東家的 NDA（保密協議），我已經將所有敏感的商業數據進行了 Anonymization（匿名化） 處理，並將核心邏輯封裝在通用模組中。目前您看到的數據是經過擾動處理的測試數據，目的是展示自動化管線的處理能力。**

我的工程核心理念在於：**將混亂且高度依賴人工的跨部門業務流程，轉化為具備防禦性、可擴展性與高容錯率的自動化系統。** 擅長處理異構系統（SAP ERP, CRM, HR 系統）的數據對齊，並透過演算法與機器學習解決實際的商業痛點。

### 核心技術棧

* **數據工程與分析：** Python (Pandas, Numpy), SQL (MSSQL, MySQL), ETL 管線設計
* **自動化與爬蟲：** Playwright, Selenium, Undetected Chromedriver, Win32COM
* **演算法與模型：** LightGBM, Random Forest, 貪婪演算法, 2-Opt, Jaccard 模糊匹配
* **架構與設計模式：** Strategy Pattern, 非同步與併發處理, 雙軌/分離式爬蟲架構

## 核心專案實績 (Featured Projects)

### 1. 全球 B2B 交易額度判定與自動化月結系統

**技術關鍵字：** `SAP ERP` `CRM API` `Strategy Pattern` `數據正規化`

* **業務痛點：** 集團跨足台灣與海外六國，各國稅率、共用信用額度與逾期判定邏輯不同，過去高度依賴財務與業務人工比對多份 Excel 與 SAP 報表，極易出錯。
* **工程決策與價值：**
* **導入策略模式 (Strategy Pattern)：** 針對多國不同的買方前綴、稅率與共用額度邏輯，設計了 `CountryConfig` 與 `BaseStrategy` 基礎類別。當擴展新國家時，僅需繼承並覆寫單一方法，徹底解決程序導向造成的技術債。
* **複雜聚合與防禦機制：** 實作母子公司共用信用池的向上聚合演算法，並建立“呆帳、逾期、無交易”的強規則攔截防線。封裝 `parse_mixed_ts` 與 `get_latest_excel` 函數，防禦歷史殘留的混亂時間戳與人為破壞的共用檔案，實現全自動的 SAP 額度變更匯入。



### 2. AI 智慧派車與路徑最佳化管線

**技術關鍵字：** `LightGBM` `Random Forest` `貪婪演算法` `2-Opt` `自然語言提取`

* **業務痛點：** 物流派車極度依賴調度員經驗，面對每日海量 B2B/B2C 訂單，人工劃分區域與決定併車順序耗時且缺乏效率。
* **工程決策與價值：**
* **雙層預測架構：** 首層利用 Random Forest 預測訂單所屬營業所與責任區；次層建構訂單配對矩陣，訓練 LightGBM 模型推論兩張訂單的(併車機率)。
* **演算法後處理兜底：** 針對 AI 模型容易產生的(低載重孤兒車次)，開發基於貪婪演算法的強制收養機制，搭配 2-Opt 進行配送路徑排序。成功降低無效派遣比例，提升各營業所車隊的平均滿載率。
* **非結構化數據拯救：** 開發客製化正則引擎，從業務隨意填寫的備註中精準提取板材重量與建材規格，挽救大量因重量缺失導致模型崩潰的訂單。



### 3. 高防護跨國數據採集與清洗引擎 (Web Scraping Architecture)

**技術關鍵字：** `Playwright` `JS Blob Fetch` `動態代理池` `Jaccard 模糊匹配`

* **業務痛點：** 人資與業務需大量監控海內外競品職缺與候選人履歷（Seek, Indeed, Boss直聘, 智聯招聘），目標網站具備極端嚴苛的反爬機制與動態 DOM 結構。
* **工程決策與價值：**
* **分離式雙軌抓取架構：** 針對智聯與 SEEK，將(列表廣泛掃描)與(詳情深層提取)解耦。一階抓取 ID 並與資料庫做差集去重，二階才進入詳情頁。有效降低無效請求，規避封鎖。
* **底層 JS 注入與人機協作：** 面對 Indeed 的履歷 Blob 下載限制，直接於瀏覽器環境執行 JavaScript 攔截 Fetch 請求轉 Base64 匯出。並實作 DOM 監控，遇到絕對防護時透過企業微信呼叫人工更新 Cookie，以半自動架構換取最高穩定度。
* **精確模糊匹配引擎：** 在 Boss 專案中，為解決地圖 API 回傳名稱與公司主檔不一致的問題，開發結合雙連字與 Jaccard 相似度的評分引擎，大幅拉升有效聯絡電話的補齊率。



### 4. 業務流轉自動化與動態負載均衡系統

**技術關鍵字：** `Subprocess 排程控制` `AST 安全解析` `動態分配演算法`

* **業務痛點：** 業務團隊每日需處理複雜的跟進名單（客訴流失、久未叫貨、展館再訪），名單派發不均且容易踩到(防打擾)紅線引發客訴。
* **工程決策與價值：**
* **第一聯絡人降級尋找演算法：** 針對 CRM 中大量離職或空號的無效聯絡人，設計 `pick_contact` 演算法。依序比對職務權重（老闆 > 總監）與關係狀態，兜底找出最有效的 09 手機號碼。
* **多維度防打擾與排重：** 透過集合運算（Set Operations）交叉比對近三個月的拜訪、K大視訊與型錄領取紀錄，並依據公司型態動態計算冷卻期。
* **隔離式中央排程器：** 放棄難以維護的系統排程，開發基於 Excel 驅動的中央排程腳本。利用 `subprocess.run` 隔離執行近四十支子腳本，內建重試退避機制與 StdErr 崩潰日誌捕獲，確保單一任務 Memory Leak 絕對不波及主流程。

## 關於我的工程思維 (Engineering Mindset)

在此倉庫的原始碼與紀錄中，你將看到我不斷在(完美架構)與(實務交付)之間做出取捨：

1. **防禦性程式設計 (Defensive Programming)：** 永遠不信任外部數據源。針對 SAP 傳來的非法字元使用 `ILLEGAL_CHARACTERS_RE` 強制洗白；針對除以零的轉換率計算設定安全閥值；針對可能被鎖死的共用 Excel 實作 PermissionError 繞過。
2. **直面技術債 (Managing Technical Debt)：** 當 CRM API 封閉且不允許直接刪除工作流中的任務時，我選擇透過逆向獲取 `procInstId` 模擬代理人進行 Withdraw 的繁瑣手段來達成目的，以解決業務的實際痛點。
3. **數據驅動的業務價值 (Business Value First)：** 所有的 ETL 與模型訓練，大部分都導向 BI 報表的視覺化監控（如漏斗圖、甘特图、熱力圖與轉換率矩陣），讓管理層能直觀看見自動化管線帶來的折損率降低與效能提升。
